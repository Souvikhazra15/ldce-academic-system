QUICK START - LOGIN PAGE
========================

WHAT'S NEW:
===========
✅ Professional login page with Google OAuth
✅ College domain email validation
✅ Persistent session management
✅ Secure authentication flow
✅ Responsive design

HOW TO ACTIVATE:
================

Step 1: Get Google Client ID
-----------------------------
1. Visit: https://console.cloud.google.com/
2. Create new project or select existing
3. Enable Google+ API
4. Go to Credentials
5. Create OAuth 2.0 Client ID (Web Application)
6. Add authorized redirect URIs:
   - http://localhost:5173
   - http://localhost:3000
   - https://yourdomain.com (production)
7. Copy the Client ID

Step 2: Update LoginPage.jsx
----------------------------
File: src/pages/LoginPage.jsx
Line: 28
Find:  <GoogleOAuthProvider clientId="YOUR_GOOGLE_CLIENT_ID_HERE">
Replace with your actual Client ID from Google Console

Step 3: Configure College Domains
---------------------------------
File: src/pages/LoginPage.jsx
Line: 10-11
Update ALLOWED_DOMAINS array:
const ALLOWED_DOMAINS = ['@yourdomain.edu', '@college.ac.in'];

Add your college email domain endings here.

Step 4: Run the App
------------------
npm start

You should see:
1. Login page with Google sign-in button
2. On login with valid college email → Dashboard loads
3. Faculty name shows in navbar
4. Logout option in user menu

WHAT HAPPENS WHEN FACULTY LOGS IN:
==================================

1. Faculty clicks "Sign in with Google"
2. Google authentication dialog opens
3. Faculty selects their Google account
4. System verifies email domain is allowed
5. If valid:
   ✓ Faculty data stored in localStorage
   ✓ Dashboard loads automatically
   ✓ Faculty name appears in navbar
   ✓ All features accessible
6. If invalid:
   ✗ Error message: "Access denied. Please use your college domain email"
   ✗ Faculty remains on login page
   ✗ Can try again with different account

PERSISTENT LOGIN:
=================
After first login:
- Faculty can close browser and reopen app
- Faculty is automatically logged in
- No need to login again
- Session persists until manual logout

LOGOUT:
======
1. Click faculty name in navbar (top right)
2. Click "Logout" option
3. Session cleared
4. Redirected to login page
5. Faculty can login again

TROUBLESHOOTING:
================

Problem: "Invalid Client ID" error
Solution: Double-check your Client ID matches exactly what's in Google Cloud

Problem: "Redirect URI mismatch" error
Solution: Add http://localhost:5173 to authorized redirect URIs in Google Cloud

Problem: "Access denied - invalid domain" message
Solution: Update ALLOWED_DOMAINS in LoginPage.jsx to include your college domain

Problem: Faculty stays logged in even after close
Solution: This is correct behavior. Use Logout button to clear session.

SECURITY NOTES:
===============
✓ Tokens validated via JWT decode
✓ College domain verification ensures only faculty can access
✓ Logout function clears all session data
✓ Tokens stored securely (can upgrade to secure cookies in production)
✓ Always verify tokens on backend in production environment
